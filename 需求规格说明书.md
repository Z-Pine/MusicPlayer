# MusicPlayer 需求规格说明书 (SRS)

> 版本：v1.0  
> 日期：2026-05-08  
> 状态：初稿

---

## 1. 项目概述

### 1.1 项目背景
开发一款面向个人使用的本地音乐播放器，主打**轻量、高效、简洁**。在提供完善的本地音乐库管理体验的同时，集成网易云音乐公开 API，用于在线搜索歌词，弥补本地歌词缺失的场景。

### 1.2 项目目标
- 构建资源占用低、响应迅速的 Windows 桌面音乐播放器
- 提供舒适的本地音乐扫描、管理与播放体验
- 支持歌词同步显示（本地 + 在线获取）
- 预留合理的扩展接口，为后续多平台（Android / iOS）迁移做技术铺垫

### 1.3 术语定义

| 术语 | 说明 |
|------|------|
| 音乐库 | 用户指定的本地目录集合，播放器从中扫描并索引音频文件 |
| 播放列表 | 用户自定义的歌曲集合，与物理文件夹解耦 |
| 最近播放 | 系统自动维护的、按播放时间倒排的歌曲队列 |
| 继续播放 | 启动应用时，自动恢复上次退出时的播放状态（歌曲、进度、播放模式） |
| 全局快捷键 | 即使播放器未聚焦，也能响应的键盘控制（如播放/暂停、切歌） |

## 2. 总体约束与假设

- **目标平台**：Windows 10/11（一期）；Android / iOS（远期评估）
- **音频来源**：以本地音频文件为主；不直接播放在线流媒体，但可调取网易云 API 获取歌词信息
- **部署方式**：桌面独立窗口应用，安装包或绿色版发布
- **资源要求**：内存占用目标控制在 150MB 以内；冷启动时间 < 2 秒
- **用户规模**：单用户自用，无需账号体系、云同步、社交功能

## 3. 功能需求

### 3.1 本地音乐库管理

| 需求 ID | 需求描述 | 优先级 |
|---------|----------|--------|
| F-001 | 允许用户添加一个或多个本地文件夹作为「音乐库源」 | P0 |
| F-002 | 自动扫描指定目录及其子目录，识别主流音频格式（见 6.1） | P0 |
| F-003 | 读取并解析音频元数据（ID3 / Vorbis Comment 等），提取标题、艺术家、专辑、封面、年份、流派 | P0 |
| F-004 | 支持手动触发「重新扫描」，检测新增、删除、修改的文件并更新索引 | P1 |
| F-005 | 播放器**不接管文件整理**，仅读取现有目录结构，不在磁盘上移动或重命名用户文件 | P0 |
| F-006 | 提供按「文件夹视图」浏览歌曲的能力（保留用户原始的目录层级） | P1 |

### 3.2 播放控制

| 需求 ID | 需求描述 | 优先级 |
|---------|----------|--------|
| F-007 | 支持基础控制：播放、暂停、上一首、下一首、进度拖拽 | P0 |
| F-008 | 支持多种播放模式：顺序播放、单曲循环、列表循环、随机播放 | P0 |
| F-009 | 支持音量调节与静音切换，记忆退出前的音量值 | P0 |
| F-010 | 支持倍速播放（如 0.5x ~ 2.0x，步进 0.1x） | P1 |
| F-011 | 支持全局快捷键 / 多媒体键盘控制（播放/暂停、上一首、下一首、音量加减） | P0 |
| F-012 | 记住上次播放进度：退出时保存当前播放的歌曲及精确到秒的进度；下次启动时提供「继续播放」入口 | P0 |

### 3.3 播放列表与队列

| 需求 ID | 需求描述 | 优先级 |
|---------|----------|--------|
| F-013 | 用户可创建、重命名、删除自定义播放列表 | P0 |
| F-014 | 支持将歌曲添加到指定播放列表，同一首歌可存在于多个列表 | P0 |
| F-015 | 提供「最近播放」自动列表，按播放时间倒序展示最近 N 首（默认 100 首，可配置） | P0 |
| F-016 | 提供「我喜欢的」快速收藏列表，支持一键星标/取消星标 | P1 |

### 3.4 歌词显示

| 需求 ID | 需求描述 | 优先级 |
|---------|----------|--------|
| F-017 | 优先加载同目录下同名歌词文件（`.lrc` / `.txt`） | P0 |
| F-018 | 若本地无歌词，自动调用网易云音乐公开 API，根据「歌曲名 + 艺术家」搜索并匹配歌词 | P0 |
| F-019 | 支持歌词同步滚动与高亮当前行 | P0 |
| F-020 | 歌词面板支持显示/隐藏切换 | P1 |
| F-021 | 若在线匹配失败，展示友好的「暂无歌词」提示 | P1 |

### 3.5 界面与交互

| 需求 ID | 需求描述 | 优先级 |
|---------|----------|--------|
| F-022 | 采用桌面窗口应用形态，主窗口包含：左侧导航、中间内容区、底部播放控制栏 | P0 |
| F-023 | 界面风格简洁，避免过度视觉元素；支持多种主题色调切换（如浅色 / 深色 / 自定义强调色） | P0 |
| F-024 | 歌曲列表支持多列排序（标题、艺术家、专辑、时长、添加时间） | P1 |
| F-025 | 支持在任务栏显示播放进度缩略图或提供任务栏媒体控制（视 Windows 版本支持） | P2 |

### 3.6 网易云音乐 API 集成（仅歌词）

| 需求 ID | 需求描述 | 优先级 |
|---------|----------|--------|
| F-026 | 集成网易云音乐公开 API，用于根据歌曲元数据检索歌词 | P0 |
| F-027 | 提供网络异常时的降级处理（提示网络错误并展示「暂无歌词」） | P1 |
| F-028 | **一期不涉及**：在线播放、歌单导入、用户登录、评论、推荐算法等 | - |

## 4. 非功能需求

### 4.1 性能需求

| 需求 ID | 需求描述 |
|---------|----------|
| NF-001 | 应用冷启动时间 ≤ 2 秒（在主流 SSD 配置的 Windows 11 设备上） |
| NF-002 | 运行时内存占用 ≤ 150 MB（播放状态下，音乐库规模 10,000 首以内） |
| NF-003 | 扫描 5,000 首歌曲的元数据应在 10 秒内完成 |
| NF-004 | 播放过程中 CPU 占用应极低（解码与渲染不应导致风扇明显加速） |

### 4.2 兼容性需求

| 需求 ID | 需求描述 |
|---------|----------|
| NF-005 | 兼容 Windows 10 版本 1903 及以上、Windows 11 |
| NF-006 | 支持高 DPI 显示（125%、150%、200% 缩放无模糊） |
| NF-007 | 响应 Windows 系统深色/浅色模式切换（或应用内独立设置） |

### 4.3 可靠性与健壮性

| 需求 ID | 需求描述 |
|---------|----------|
| NF-008 | 单首歌曲文件损坏或元数据异常时，跳过该文件并记录日志，不应导致扫描崩溃 |
| NF-009 | 音乐库源文件夹被移动/删除时，优雅标记文件为「不可用」，而非直接报错退出 |
| NF-010 | 网络请求（网易云 API）需设置超时（建议 5 秒），避免长时间阻塞 UI |

### 4.4 可扩展性

| 需求 ID | 需求描述 |
|---------|----------|
| NF-011 | 架构层面预留音频解码引擎抽象层，便于后续增加格式支持 |
| NF-012 | 数据存储层设计应考虑后续跨平台（Android / iOS）的数据结构兼容性 |
| NF-013 | 在线模块与核心播放模块解耦，便于未来增加更多在线数据源或完全移除 |

## 5. 技术选型建议

基于「**执行效率高、系统资源占用少**」的核心诉求，推荐以下两种技术路线：

### 方案 A：Tauri (Rust + Web 前端) —— 推荐
- **优势**：二进制体积极小（通常 < 10MB），内存占用远低于 Electron，性能接近原生；可复用现代前端技术栈做 UI，开发效率较高
- **劣势**：生态较新，部分 Windows 原生 API 需编写 Rust 插件
- **适用**：追求极致轻量、熟悉 Web 技术的开发者

### 方案 B：Qt (C++)
- **优势**：真正的跨平台原生框架，性能极佳，Windows 下表现非常成熟，多媒体模块完善
- **劣势**：C++ 开发门槛较高，编译配置相对复杂
- **适用**：对 C++ 熟悉、追求原生性能且不考虑大量 Web 技术复用的开发者

### 方案 C：WinUI 3 / Windows App SDK
- **优势**：微软官方推荐的最新 Windows 原生 UI，与系统风格高度一致，性能优秀
- **劣势**：跨平台能力弱（不利于后续 Android/iOS 扩展），生态相对较新
- **适用**：只专注 Windows、希望最原生体验的开发者

> **建议**：若希望长期可能扩展到移动端，优先 **方案 A (Tauri)**；若专注 Windows 且追求绝对原生性能，优先 **方案 B (Qt)**。

## 6. 数据与格式规范

### 6.1 支持的音频格式（一期）

- MP3 (`.mp3`)
- FLAC (`.flac`)
- WAV (`.wav`)
- AAC / M4A (`.aac`, `.m4a`)
- OGG Vorbis (`.ogg`)

> 暂不支持 CUE 分轨文件。

### 6.2 元数据字段

播放器应索引并展示的字段：
- 文件路径（内部标识）
- 标题 (Title)
- 艺术家 (Artist) —— 支持多艺术家
- 专辑 (Album)
- 专辑艺术家 (Album Artist)
- 年份 (Year)
- 流派 (Genre)
- 时长 (Duration)
- 封面图 (Cover Art)
- 比特率 / 采样率（可选展示）

### 6.3 数据持久化

| 数据类型 | 建议存储方式 |
|----------|--------------|
| 音乐库索引、歌曲元数据 | SQLite 本地数据库 |
| 播放列表、用户配置、主题设置 | SQLite 或 JSON/YAML 配置文件 |
| 播放进度、最近播放记录 | SQLite |
| 缓存的在线歌词 | 本地文本缓存（.lrc 文件或 SQLite） |

## 7. 界面结构草案

```
┌──────────────────────────────────────────────────────┐
│ [Menu]  MusicPlayer                           [_][X] │  <- 标题栏
├──────────┬───────────────────────────────────────────┤
│          │                                           │
│  音乐库   │                                           │
│  文件夹   │          主内容区                         │
│  播放列表 │          （歌曲列表 / 歌词面板）            │
│  最近播放 │                                           │
│  设置    │                                           │
│          │                                           │
├──────────┴───────────────────────────────────────────┤
│  [封面]  歌曲名 - 艺术家      ◀  ⏸  ▶  [进度条]  🔊  │  <- 播放控制栏
└──────────────────────────────────────────────────────┘
```

### 7.1 主要视图
1. **歌曲列表视图**：展示当前音乐库 / 文件夹 / 播放列表内的歌曲，支持排序、搜索、右键菜单
2. **歌词视图**：全屏或侧边展示同步滚动歌词
3. **设置视图**：音乐库源管理、主题切换、快捷键绑定、扫描策略

## 8. 里程碑与迭代计划

| 阶段 | 目标 | 核心交付 |
|------|------|----------|
| **MVP** | 可播放本地音乐 | 本地扫描、基础播放控制、播放列表、最近播放、记住进度 |
| **v0.2** | 完善的音乐管理体验 | 文件夹视图、排序搜索、元数据编辑（可选）、倍速播放 |
| **v0.3** | 歌词体验闭环 | 本地歌词加载、网易云 API 歌词搜索、歌词同步高亮 |
| **v0.4** | 体验打磨 | 全局快捷键、主题系统、高 DPI 适配、安装包制作 |
| **v1.0** | 稳定发布 | Bug 修复、性能优化、文档完善、长期维护 |
| **远期** | 跨平台评估 | 根据使用反馈，评估 Android / iOS 版本的可行性 |

## 9. 风险与注意事项

| 风险项 | 影响 | 缓解措施 |
|--------|------|----------|
| 网易云 API 不稳定或策略变更 | 在线歌词功能失效 | 做好本地歌词优先策略；预留切换其他歌词源的接口 |
| 本地音乐库规模极大（>10万首） | 扫描与查询性能下降 | 采用异步扫描 + 增量更新；数据库索引优化 |
| 音频元数据乱码 / 不规范 | 歌曲信息显示异常 | 支持多编码探测（UTF-8 / GBK）；允许用户手动修正 |
| Windows 多媒体键冲突 | 全局快捷键失效 | 提供可自定义的快捷键绑定；避免与常见软件冲突 |
