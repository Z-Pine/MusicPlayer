# 歌单删除按钮无响应 Bugfix 需求文档

## 简介

在歌单的歌曲列表界面（PlaylistView.vue），当用户点击歌曲右侧的删除按钮（❌）时，如果该歌曲未被选中，删除操作不会执行，按钮无响应。这是因为 `removeSongs` 函数依赖 `selectedSongIds` 集合，当集合为空时会直接返回，导致单个歌曲的删除操作失败。

**影响范围**：所有歌单的歌曲删除功能（不包括"最近播放"列表）

**文件位置**：`f:\VSCode Projects\MusicPlayer\src\views\PlaylistView.vue`

## Bug 分析

### 当前行为（缺陷）

1.1 WHEN 用户点击未选中歌曲右侧的删除按钮（❌）THEN 系统不执行任何操作，按钮无响应

1.2 WHEN 用户点击未选中歌曲右侧的删除按钮（❌）THEN 系统不显示删除确认对话框

1.3 WHEN `selectedSongIds` 集合为空时调用 `removeSongs` 函数 THEN 函数直接返回，不执行删除逻辑

### 预期行为（正确）

2.1 WHEN 用户点击任意歌曲右侧的删除按钮（❌）THEN 系统应该删除该歌曲，无论该歌曲是否被选中

2.2 WHEN 用户点击单个歌曲的删除按钮（❌）THEN 系统应该显示删除确认对话框，提示删除 1 首歌曲

2.3 WHEN 删除按钮接收到歌曲 ID 参数时 THEN `removeSongs` 函数应该删除指定的歌曲，而不是依赖 `selectedSongIds` 集合

2.4 WHEN 删除按钮未接收到歌曲 ID 参数时 THEN `removeSongs` 函数应该删除所有选中的歌曲（批量删除功能）

### 不变行为（回归预防）

3.1 WHEN 用户选中多首歌曲后点击任意一首的删除按钮 THEN 系统应该继续删除所有选中的歌曲（批量删除功能保持不变）

3.2 WHEN 用户通过右键菜单选择"从歌单移除"THEN 系统应该继续正常删除选中的歌曲

3.3 WHEN 用户在"最近播放"列表中 THEN 删除按钮应该继续保持隐藏状态

3.4 WHEN 删除操作成功后 THEN 系统应该继续显示 Toast 提示消息，刷新歌曲列表，并清空选中状态

3.5 WHEN 用户取消删除确认对话框 THEN 系统应该继续取消删除操作，不修改歌单
